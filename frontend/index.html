Speculative Prefill: Eliminating Dead Air in Voice Agents

What if your LLM started thinking before you finished speaking? Input-time speculative decoding overlaps inference with speech, cutting voice-to-voice latency from seconds to milliseconds.

The Problem

Typical voice agents run a sequential pipeline: wait for the user to finish speaking, transcribe the full utterance, send it to the LLM, wait for the full response, then synthesize speech. Each stage blocks the next. For a 14B parameter model, this means 2–4 seconds of dead air after the user stops talking.

The Insight

PredGen starts LLM inference while the user is still speaking. As partial transcripts stream in from ASR, the model speculatively generates tokens. When the final transcript arrives, most of the generation is already done — only a small correction step remains. The result: near-zero latency after speech ends.
